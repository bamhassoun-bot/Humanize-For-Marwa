How to Use humanize
Complete guide · 5 languages · 35+ engines

Quick Start (3 steps)

1️⃣
Paste your AI text into the input box. Language is detected automatically — or pick one from the dropdown. You'll see live word/sentence counts appear.
2️⃣
Click "clean & humanize" (or press ⌘↵ / Ctrl↵). The tool runs all active engines and shows the cleaned output below.
3️⃣
Review the output tab — read, edit, then copy, download, or run again if needed.

Getting the Best Results

  • Start with Moderate aggressiveness (default). Light mode only does tier-1 swaps; Aggressive does all three tiers and may over-simplify technical content.
  • Use "Scan only" first on important text. It shows you the AI-risk score and pattern badges without changing anything.
  • Run it twice using "run again" — the second pass catches patterns the first pass missed, especially after structural changes.
  • Toggle off registers you don't need. If your text is NOT medical, turn off the Medical register to avoid replacing legitimate terms like "cardiac" or "renal."
  • For Arabic text: the tool auto-switches to RTL layout. Turn off "Add contractions," "Split long sentences," and "Numerals → words" — those engines are English-only.
  • Use the Diff tab to see exactly what changed. Word diff highlights each swapped word; char diff shows character-level edits.
  • Add Custom Rules for your own recurring terms. Supports plain text or /regex/gi syntax.

Understanding the Engines

  • Core tab: The most impactful engines. AI filler (700+/lang), vocabulary simplification (1000+/lang), contractions, AI formatting, and the new Double Negatives, Weasel Words, Meta-Commentary, and AI Signatures engines.
  • Voice tab: Reduces passive voice, hedging, weak verbs, hollow adverbs, and expletive constructs. "First-person voice" shifts "one must" → "you must." "Casual tone" replaces formal greetings.
  • Style tab: Varies sentence openers, splits long sentences, reduces repetition. "Rhetorical punches" and "em-dashes" add stylistic texture — use sparingly.
  • Structure tab: Converts bullet lists → prose, removes AI markdown headers and bold, fixes paragraph breaks.
  • Registers tab: Corporate, Academic, Legal, Medical, Tech, Business, Educational — turn off any registers that match your actual domain.
  • Custom tab: Your own find → replace rules, run after all engines.

Reading the Scores

  • AI Risk Score (0–100): Based on pattern matching. Below 30 = low risk, 30–60 = moderate, above 60 = likely AI-generated. A good target is under 25.
  • Flesch Score: Readability index. 60–80 = standard to easy. Higher is more readable. After cleaning, this should go up.
  • Tone Profile: Shows formal / conversational / academic balance. After humanizing, "conversational" should rise.
  • AI Source Detector: Fingerprints ChatGPT, Claude, Gemini, Copilot, Grok. Only reliable when the text contains model-specific phrases.

Output Tabs Explained

  • output: The cleaned text. Editable. Use A− / A+ to adjust font size.
  • stats: Before/after comparison of word count, sentence length, Flesch, passive density, AI risk.
  • tone: Formal / Conversational / Academic before and after.
  • changes: Per-engine hit counts — shows exactly which engines fired.
  • diff: Visual word-level diff (red = removed, green = added).
  • side-by-side: Original and cleaned text next to each other.
  • heatmap: Word frequency heat — red = highly repeated, yellow = medium, green = low.
  • history: Last 8 runs. Click any entry to reload it.

Keyboard Shortcuts

  • ⌘ ↵ / Ctrl ↵ — Run humanize
  • ⌘ Z / Ctrl Z — Undo last run
  • Esc — Clear all

Which Language Mode to Use

Leave the selector on 🌐 auto — detection is accurate for texts over 50 words. Override manually if you're pasting short fragments or mixed-language text. Arabic is detected by character range (Unicode Arabic block), so it's very reliable.

The Best Workflow in Practice

1️⃣
Run it once — then open the diff tab to see every change the tool made.
2️⃣
Manually fix anything that reads wrong. The output tab is editable — rewrite any swap that sounds off or changed your meaning.
3️⃣
Run it again on the edited version using re-humanize. The second pass catches patterns the first missed, especially after structural changes.
4️⃣
Do a final read-aloud. Your ear catches what the tool can't — rhythm problems, unnatural phrasing, anything that still sounds generated.
